# 股价智能缓存系统 v2.1

## 🚀 新增功能

### 1. 多数据源自动切换
- **新浪财经**：主数据源，最快（129ms）
- **腾讯财经**：备用，港股支持好
- **东方财富**：备用，数据全但响应慢
- 自动切换：主数据源失败自动切换备用源

### 2. 缓存预热
- 开盘前自动加载常用股票缓存
- 支持 `--warmup` 命令手动预热

### 3. 常用股票管理
- 添加常用股票：`--add <代码>`
- 移除常用股票：`--remove <代码>`
- 查看常用股票：`--favorites`

### 4. 自动清理过期缓存
- 定期清理7天前的缓存文件
- 支持 `--cleanup` 命令手动清理

### 5. 统计监控
- 实时查看缓存命中率
- 统计总查询次数、缓存命中、API调用

**熔断机制**：

```python
# 数据源熔断规则

    'half_open_requests': 1,     # 恢复时先尝试1次

# 示例：新浪财经连续失败3次
# → 熔断10分钟，期间优先使用腾讯财经
# → 10分钟后尝试1次，成功则恢复，失败则继续熔断

## 查询流程（阶段划分 + 检查点）

### 阶段一：缓存检查

```
📍 检查点 1.1 - 缓存状态检测
├─ 输入：股票代码
├─ 操作：检查 ~/.hermes/stock_cache/{code}.json 是否存在
├─ 判断：
│   ├─ 缓存不存在 → 进入阶段二（数据获取）
│   └─ 缓存存在 → 进入检查点 1.2
│
📍 检查点 1.2 - 缓存时效验证
├─ 读取缓存中的 cache_time 和 trading_status
├─ 根据当前时间判断有效期：
│   ├─ 盘中（9:30-15:00）：有效期 5 分钟
│   ├─ 盘后（15:00后）：有效期 24 小时
│   └─ 周末/节假日：有效期至下一交易日
├─ 判断：
│   ├─ 缓存有效 → 返回缓存数据 ✓
│   └─ 缓存过期 → 进入阶段二（数据获取）
```

### 阶段二：数据获取（多数据源切换）

```
📍 检查点 2.1 - 主数据源尝试（新浪财经）
├─ 请求：https://hq.sinajs.cn/list={code}
├─ 超时设置：3秒
├─ 结果判断：
│   ├─ 成功（HTTP 200 + 有效数据）→ 进入阶段三 ✓
│   ├─ 超时 → 记录日志，进入检查点 2.2 ⚠️
│   └─ 失败（非200/数据异常）→ 进入检查点 2.2 ⚠️
│
📍 检查点 2.2 - 备用数据源A（腾讯财经）
├─ 触发条件：新浪失败/超时
├─ 请求：https://web.sqt.gtimg.cn/q={code}
├─ 超时设置：5秒
├─ 结果判断：
│   ├─ 成功 → 进入阶段三 ✓
│   └─ 失败 → 进入检查点 2.3 ⚠️
│
📍 检查点 2.3 - 备用数据源B（东方财富）
├─ 触发条件：腾讯失败
├─ 请求：https://push2.eastmoney.com/api/qt/stock/get
├─ 超时设置：5秒
├─ 结果判断：
│   ├─ 成功 → 进入阶段三 ✓
│   └─ 失败 → 进入检查点 2.4（降级处理）⚠️
│
📍 检查点 2.4 - 降级处理（所有数据源失败）
├─ 触发条件：所有数据源均失败
├─ 处理策略：
│   ├─ 策略A：使用过期缓存（标记为"延迟数据"）
│   │   └─ 返回缓存数据 + 警告提示
│   └─ 策略B：无缓存时返回错误
│       └─ 返回 {error: "所有数据源不可用，请稍后重试"}
└─ 记录错误日志到 ~/.hermes/logs/stock_cache_errors.log
```

### 阶段三：缓存更新

```
📍 检查点 3.1 - 数据标准化
├─ 统一字段格式（name, price, last_close, etc.）
├─ 添加元数据：
│   ├─ cache_time: 缓存时间戳
│   ├─ trading_status: 交易状态
│   └─ source: 数据来源
│
📍 检查点 3.2 - 写入缓存
├─ 写入 ~/.hermes/stock_cache/{code}.json

## 缓存失效异常处理

### 异常类型与处理策略

| 异常类型 | 触发条件 | 处理策略 | 用户提示 |
|---------|---------|---------|---------|
| **缓存文件损坏** | JSON解析失败 | 删除损坏文件，重新获取 | 无（静默处理） |
| **缓存目录不存在** | 首次使用 | 自动创建目录 | 无 |
| **权限错误** | 写入失败 | 使用/tmp作为备用 | 警告：缓存功能受限 |
| **磁盘空间不足** | 写入失败 | 清理最旧缓存 | 警告：已清理旧缓存 |
| **网络超时** | 数据源无响应 | 切换备用源 | 显示当前尝试的数据源 |
| **所有数据源不可用** | 全部失败 | 使用过期缓存或报错 | 明确提示数据可能延迟 |

## 常见问题

### Q: 如何判断缓存是否有效？

A: 系统自动判断，根据当前交易状态决定缓存有效期：
- 盘中：5分钟
- 盘后：24小时
- 周末/节假日：直到下一个交易日

### Q: 数据源会自动切换吗？

A: 会，主数据源（新浪）失败时自动切换到腾讯、东方财富。每次切换都有检查点记录。

### Q: 如果所有数据源都失败怎么办？

A: 系统会尝试使用过期缓存（并明确提示数据可能延迟），如果连缓存都没有，则返回错误信息。

### Q: 缓存数据会自动更新吗？

A: 会，当缓存过期时自动获取新数据并更新缓存。

### Q: 如何强制更新缓存？

A: 使用 `--force` 参数或调用 `get_stock_price(code, force_update=True)`

### Q: 缓存文件会占用多少空间？

A: 每只股票约1KB，1000只股票也仅占1MB空间。

### Q: 如何添加常用股票？

A: 使用 `--add <代码>` 命令，常用股票会在缓存预热时自动加载。

### Q: 如何查看错误日志？

A: 使用 `--errors` 参数查看最近的错误记录，或直接查看 `~/.hermes/logs/stock_cache_errors.log`。
