# GitHub 知识追踪器

从 GitHub 仓库提取知识内容，保存到长期记忆，并设置定时任务自动追踪更新。

## 适用场景

- 追踪提示词库仓库更新（如 awesome-gptimage2）
- 维护模板库、最佳实践库等动态更新资源
- 监控开源项目的文档变化并自动同步

## 操作流程

### 第一步：提取知识库内容

1. 访问仓库 README 和主要文档
2. 结构化提取核心内容：
   - 核心概念/公式/框架
   - 分类体系
   - 可复用模板
   - 高级技巧
   - 注意事项/避坑指南
3. 记录仓库地址和当前更新日期

### 第二步：压缩优化记忆格式

将内容高度压缩以适应 memory 空间限制（单条 2200 chars）：

```markdown
【知识库名称】
仓库：URL
更新：YYYY-MM-DD

核心概念：[要点1] / [要点2] / [要点3]
神关键词：[关键术语]
模式：[模式A] / [模式B]

分类：[分类1] / [分类2] / [分类3]...

原则：[原则1]；[原则2]；[原则3]

核心模板：
1. [模板名称]："模板内容"
2. [模板名称]："模板内容"
...

高级技巧：[技巧1]；[技巧2]；[技巧3]
```

压缩原则：
- 使用 `/` 代替换行分隔同类项
- 使用 `；` 分隔短句
- 删除冗余描述，保留最核心关键词
- 模板内容保留可直接复制的最小有效版本

### 第三步：清理 memory 空间

如果 memory 已满（> 2000/2200 chars），按优先级删除：
1. 临时任务记录（如安装问题解决、单次调试记录）
2. 过时的市场数据、已过期的信息
3. 低优先级的项目记录
4. 保留：用户偏好设置、长期项目核心设定、关键配置参数

### 第四步：设置定时更新任务

使用 cronjob 创建定期检查任务：

```yaml
name: "[知识库名称] 更新追踪"
schedule: "0 10 * * 1"  # 每周一 10:00
prompt: |
  请检查 GitHub 仓库 [URL] 的最新更新。
  1. 查看最近提交记录和更新日期
  2. 对比记忆中保存的版本（[当前日期]）
  3. 如果有更新，提取新增内容
  4. 将更新摘要发送到微信
  5. 更新 memory 中的对应条目
deliver: "origin"
```

频率建议：
- 活跃更新仓库：每周一次
- 稳定维护仓库：每两周一次
- 归档仓库：每月一次

### 第五步：使用知识库

当用户询问相关问题时：
1. 从 memory 中检索对应知识库条目
2. 根据用户场景匹配对应模板
3. 填入用户提供的参数（主题、名称、特征等）
4. 输出完整可直接复制的提示词

## 记忆管理最佳实践

- 单条知识库控制在 800 chars 以内，给其他内容预留空间
- 优先保存"可以直接复制使用"的模板，而不是理论描述
- 定期（每月）清理一次过期和低价值条目
- 重要但内容过多的资源：创建技能文件（SKILL.md）而不是 memory

## 常见问题

**Q: memory 总是满怎么办？**
A: 进一步压缩内容，只保留 Top 5 最常用模板，其余放入技能文件。

**Q: 如何处理非常大的仓库？**
A: 创建独立 skill 文件存放完整内容，memory 中只保存索引和 Top 3 核心模板。

**Q: cronjob 报错如何处理？**
A: 使用 `every 7d` 格式替代 cron 表达式，兼容性更好。
