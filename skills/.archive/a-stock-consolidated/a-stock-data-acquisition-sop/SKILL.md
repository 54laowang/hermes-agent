# A股数据获取SOP（时间锚定原则）

> **强制规则**：这是所有A股分析的前置步骤，优先级高于一切Skill和分析框架。

## 第二步：判断市场状态

| 时间段 | 市场状态 | 数据类型 |
|--------|----------|----------|
| 9:30前 | 盘前 | 昨日收盘 + 隔夜外盘 |
| 9:30-11:30 | 早盘盘中 | 实时行情（不推荐分析） |
| **11:30** | **午盘收盘** | **午评数据** |
| 11:30-13:00 | 午间休盘 | 午盘数据（此时可用） |
| 13:00-15:00 | 午后盘中 | 实时行情（不推荐分析） |
| **15:00后** | **收盘** | **收评数据** |
| 周末 | 休市 | 上周五收盘数据 |

## 第三步：精确搜索关键词

**必须包含三要素：日期 + 时间节点 + 数据类型**

```
"午评 {M月D日} 沪指 创业板指"

```

## 第五步：验证数据时间戳

**强制验证项：**
- [ ] 返回数据的日期是否 = 今天？
- [ ] 返回数据的时间节点是否 = 当前市场状态？

**如果验证失败：**
1. 立即停止分析
2. 重新搜索（换关键词/换数据源）
3. 禁止用错误时间的数据凑数

---

## 第六步：从搜索结果提取历史数据的强制验证

**适用场景**：当分析个股历史走势、龙虎榜、财报等历史数据时。

**三重验证原则**（必须逐项回答）：

### 6.1 时间验证
- [ ] **这是什么时间的数据？**
  - 明确日期：YYYY年M月D日
  - 明确时段：早盘/午盘/收盘/盘后
  - 验证方法：查看搜索结果的发布时间或文章标题中的日期

### 6.2 性质验证
- [ ] **数据性质是什么？**
  - 涨跌幅：具体数值（如 +10.01% 涨停 / -10.01% 跌停）
  - 数据类型：涨停/跌停/涨/跌/平
  - 验证方法：查看搜索结果正文，确认涨跌幅，不能只看标题

### 6.3 来源验证
- [ ] **数据来源是否可靠？**
  - 来源类型：官方公告/权威媒体/交易所数据/股吧讨论
  - 可靠性评级：P0（官方）> P1（权威媒体）> P2（自媒体）
  - 验证方法：交叉验证，优先使用官方数据

### 典型错误案例（2026-05-01）

**错误示范**：
```
搜索："克来机电 2026年3月 龙虎榜"
搜索结果标题："龙虎榜：克来机电晋级五连板"
错误陈述："2026年3月克来机电5连板"

实际数据：
- 时间：2026年3月12日
- 性质：跌停（-10.01%），不是涨停
- 标题中的"晋级五连板"是其他时间的新闻，被错误关联
```

**正确做法**：
1. 点击搜索结果，查看正文
2. 确认日期：2026年3月12日
3. 确认涨跌幅：-10.01%（跌停）
4. 确认龙虎榜性质：跌停龙虎榜，不是涨停榜
5. 准确陈述："2026年3月12日克来机电跌停，龙虎榜显示..."

## 常见错误

| 错误行为 | 后果 | 正确做法 |
|----------|------|----------|
| 用11:05的盘中快讯代替11:30午盘数据 | 数据不准确 | 等11:30后搜索午评 |
| 用昨日收盘数据代替今日午盘 | 时间错误 | 明确搜索"今日午评" |
| 搜索"A股今日行情" | 返回模糊结果 | 明确搜索"午间收盘+日期" |
| 不验证数据时间戳 | 引用过时数据 | 检查数据发布时间 |
