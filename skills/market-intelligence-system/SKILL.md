# market-intelligence-system

市场智能监控系统 - 整合自 `daily-market-brief` (291行) + `supervisor-mode-auto-trigger` (400行)，采用达尔文进化方法论。

## 核心定位

**监控预警层**：全市场智能监控、自动触发、简报生成、推送通知，与交易执行层协同工作。

## 架构设计

### 六大核心模块

```
┌─────────────────────────────────────────────────────────┐
│              market-intelligence-system                  │
├─────────────────────────────────────────────────────────┤
│  1. MarketMonitor       市场监控模块                     │
│  2. AutoTrigger         自动触发模块                     │
│  3. BriefGenerator      简报生成模块                     │
│  4. PushNotifier        推送通知模块                     │
│  5. DataSourceManager   数据源管理模块                   │
│  6. TimeAnchor          时间锚定模块                     │
└─────────────────────────────────────────────────────────┘
```

## 模块一：MarketMonitor（市场监控模块）

### 核心职责

- 监控A股核心指标（指数、北向、成交、涨跌停）
- 监控美股核心指标（道琼斯、纳斯达克、标普500）
- 监控连板股、人气票
- 异常波动检测

### A股监控指标

| 指标类别 | 具体内容 | 数据源 |
|---------|---------|--------|
| **指数** | 上证指数、深证成指、创业板指 | 东方财富 |
| **资金** | 北向资金净流入 | 财联社 |
| **成交** | 两市成交额 | 东方财富 |
| **情绪** | 涨停家数、跌停家数 | 东方财富 |
| **连板** | 高度板、晋级率 | 同花顺 |
| **人气** | 同花顺人气榜前三、东财飙升榜前二 | 同花顺、东方财富 |

### 美股监控指标

| 指标类别 | 具体内容 | 数据源 |
|---------|---------|--------|
| **指数** | 道琼斯、纳斯达克、标普500 | Yahoo Finance |
| **期货** | 道指期货、纳指期货 | Trading Economics |
| **VIX** | 恐慌指数 | CBOE |

### ✅ 检查点（5个）

| 检查点 | 检查内容 | 通过标准 |
|--------|---------|---------|
| **CP-MONITOR-01** | 数据源响应状态 | HTTP 200 |
| **CP-MONITOR-02** | 数据时间戳新鲜度 | 时间戳<5分钟 |
| **CP-MONITOR-03** | 指数数值有效性 | 指数>0 |
| **CP-MONITOR-04** | 北向资金数据完整性 | 数据存在且非空 |
| **CP-MONITOR-05** | 涨跌停家数一致性 | 涨停+跌停+平盘≤总数 |

### ⚠️ 异常处理（4个）

| 异常场景 | 处理策略 | 日志级别 |
|---------|---------|---------|
| **数据源超时** | 切换备用数据源 | WARNING |
| **数据缺失** | 标注缺失字段 + 继续执行 | WARNING |
| **数值异常** | 多源验证 + 记录警告 | WARNING |
| **所有数据源失败** | 推送错误通知 | ERROR |

### 🔲 边界条件（6个）

| 边界条件 | 触发场景 | 处理方式 |
|---------|---------|---------|
| **BC-MONITOR-01** | 开盘前无实时数据 | 使用昨日收盘数据 |
| **BC-MONITOR-02** | 收盘后数据冻结 | 使用收盘数据 |
| **BC-MONITOR-03** | 节假日无数据 | 输出休市简报 |
| **BC-MONITOR-04** | 涨跌停家数=0（熔断/异常） | 标注异常，推送警报 |
| **BC-MONITOR-05** | 北向资金数据延迟（盘后才公布） | 标注"盘后公布" |
| **BC-MONITOR-06** | 美股盘中（A股夜间）数据获取 | 正常获取，标注时区 |

## 模块二：AutoTrigger（自动触发模块）

### 核心职责

- 关键词匹配（美股/A股关键词）
- 市场识别（us_stock/a_stock）
- 时间锚定计算（北京时间 ↔ 美东时间）
- 市场状态判断（盘前/盘中/收盘/盘后）
- 系统提示注入

### 触发关键词

#### 美股关键词（5组正则）

```python
US_STOCK_PATTERNS = [
    r"美股.*(走势|收盘|行情|分析)",
    r"(道琼斯|纳斯达克|标普500|S&P)",
    r"(Fed|美联储|利率决议)",
    r"(Dow|Nasdaq|S&P)",
    r"earnings|财报"
]
```

#### A股关键词（3组正则）

```python
A_STOCK_PATTERNS = [
    r"A股.*(走势|收盘|行情|分析)",
    r"(上证指数|深证成指|创业板)",
    r"(财联社|东方财富)"
]
```

### 市场状态判断逻辑

#### 美股状态

| ≥ 16:00 | 收盘/盘后 | 收评数据 |

| 北京时间 | 状态 | 数据类型 |

| 11:30-13:00 | 午间休市 | 早盘收盘 |
| 13:00-15:00 | 午盘 | 午盘数据 |

## 模块三：BriefGenerator（简报生成模块）

### 核心职责

- 生成早盘/收盘/休市三种模板
- 遵循"爱在冰川"分析风格
- 周期判断（上升/震荡/退潮/冰点）
- 资金行为分析
- 情绪定位

### 三种简报模板

#### 1. 早盘简报（11:35）

**关注点**：
- 上午资金态度
- 下午看点
- 板块轮动

**数据源**：
- 早盘指数走势
- 北向资金流向
- 涨停家数

#### 2. 收盘简报（15:05）

**关注点**：
- 完整复盘
- 明日看点
- 风险提示

**数据源**：
- 全天指数走势
- 北向资金全天流向
- 涨跌停家数
- 连板股统计
- 人气热度票

#### 3. 休市简报（节假日）

**关注点**：
- 节后关注点
- 复市倒计时
- 外围市场回顾

**数据源**：
- 节前最后交易日数据
- 外围市场数据

### "爱在冰川"分析风格

#### ✅ 必须说

| 要素 | 示例 |
|------|------|
| **资金位置** | "资金在3200点做了什么选择" |
| **周期判断** | "是主升加速还是高位分歧" |
| **情绪定位** | "情绪在周期的哪个位置" |
| **性价比** | "当前位置性价比如何" |

## 模块六：TimeAnchor（时间锚定模块）

### 核心职责

- 建立时间锚点（当前时间、目标数据日期）
- 时区换算（北京时间 ↔ 美东时间）
- 市场状态判断
- 数据日期验证

### 时区换算规则

```python
# 北京时间 → 美东时间（夏令时）
美东时间 = 北京时间 - 12小时

# 北京时间 → 美东时间（冬令时）
美东时间 = 北京时间 - 13小时

# 夏令时判断
夏令时 = 3月第二个周日 → 11月第一个周日
```

## 数据流架构

### 正常流程

```
1. 用户消息/定时触发 → AutoTrigger.detect()
2. 建立时间锚点 → TimeAnchor.establish()
3. 获取市场数据 → MarketMonitor.fetch()
4. 生成简报 → BriefGenerator.generate()
5. 推送通知 → PushNotifier.send()
```

### 异常流程

```
数据源失败 → DataSourceManager.switch() → 重试
时间计算错误 → TimeAnchor.fallback() → 使用UTC
简报生成失败 → BriefGenerator.partial() → 部分简报
推送失败 → PushNotifier.retry() → 重试3次
```
