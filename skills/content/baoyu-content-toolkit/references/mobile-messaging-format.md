# 移动端消息格式最佳实践

## 背景

用户反馈：飞书消息在手机上"看起来有点凌乱"，需要简化格式以提升移动端阅读体验。

---

## 飞书/微信等平台的 Markdown 限制

### 支持的格式

- ✅ 标题（# ## ###）
- ✅ 粗体/斜体
- ✅ 列表（有序/无序）
- ✅ 代码块
- ✅ 引用块
- ✅ 链接

### 不支持/显示不佳的格式

- ❌ 表格复杂样式（手机上横向滚动）
- ❌ 多列布局
- ❌ 颜色自定义
- ❌ 字体大小调整
- ❌ 图文混排高级样式
- ❌ 表情符号有限

---

## 移动端友好的格式原则

### 1. 减少表格，多用列表

**❌ 不推荐：**

```
| 维度 | OpenAI | DeepSeek |
|------|--------|----------|
| 收入模式 | 订阅 | 行业定制 |
```

**✅ 推荐：**

```
维度对比：
- OpenAI：订阅 + API
- DeepSeek：行业定制 + 算力租赁
```

### 2. 控制每条消息长度

- 每条消息控制在 500 字以内
- 超过 500 字分多次发送
- 或生成文件，飞书只发摘要

### 3. 关键结论放最前面

**❌ 不推荐：** 先铺垫背景、数据、分析过程，最后才给结论

**✅ 推荐：** 结论先行，再展开细节

### 4. 扁平化结构

**❌ 不推荐：**

```
一、大主题
  1. 子主题
    a) 细节1
    b) 细节2
```

**✅ 推荐：**

```
一、大主题

子主题：
- 细节1
- 细节2
```

### 5. 避免嵌套过深

- 列表嵌套不超过 2 层
- 表格尽量简单，不超过 3 列
- 代码块控制在 20 行以内

---

## 平台差异

| 平台 | 表格支持 | 换行处理 | 特殊注意 |
|------|---------|---------|---------|
| 飞书 | 基础 | 需双空格 | 手机端表格横向滚动 |
| 微信 | 不支持 | 自动换行 | 避免使用表格 |
| 企微 | 基础 | 需双空格 | 同飞书 |

## 实战案例

### 案例1：多新闻摘要

**原始格式（表格）：**

```
| 新闻 | 核心事件 | 影响 |
|------|---------|------|
| OpenAI | 亏损140亿 | 商业压力 |
```

**优化后（列表）：**

```
一、OpenAI 业绩挣扎

核心问题：烧钱换增长逼近极限。
关键数据：2026年亏损预测140亿美元。
深层解读：开源倒逼闭源降价...
```

### 案例2：数据对比

**原始格式（表格）：**

```
| 时间 | 价格 | 降幅 |
|------|------|------|
| 2026 | $10 | 基准 |
| 2027 | $1 | 90% |
```

**优化后（要点列表）：**

```
Token 价格趋势：
- 2026年：$10/百万tokens（基准）
- 2027年：$1/百万tokens（降90%）
- 2028年：$0.1/百万tokens（再降90%）
```
