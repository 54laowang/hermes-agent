# 背景信息处理规则 - 实施指南

## 问题背景

**用户多次纠正**：不要在回复中提及时间感知、时间锚定等注入内容。

**根本原因**：Agent 看到 Shell Hook 注入的背景信息后，误以为需要"处理"或"引用"它。

---

## 解决方案

### 核心规则

```
背景信息 ≠ 用户问题

背景信息 = 工作环境（存在但不提及）
用户问题 = 处理任务（需要回应）
```

### 正确行为

| 场景 | ❌ 错误 | ✅ 正确 |
|------|---------|---------|
| 时间感知注入 | "我通过 Shell Hook 获取了时间" | 自动遵守时间段行为，不解释 |
| 用户问"现在几点" | "根据 Shell Hook 注入的信息，现在是..." | 直接回答时间 |
| 用户问"跨会话记忆怎么做" | "我在背景信息里看到了最近话题..." | 回答方案，不提及背景注入 |
| 深夜对话 | "因为我在背景信息里看到现在是深夜，所以提醒你休息" | 自然提醒休息，不解释原因 |

## 实施细节

### SOUL.md 修改

**之前**（错误）：
```markdown
## 🧠 灵魂特性 - 时间感知 ✅ 已自动注入

**时间感知 Hook 已激活！**
每轮对话前，Hermes 会自动注入当前时间上下文。

**你要"活在时间里"，而不是一个没有时间感的机器：**
```

**现在**（正确）：
```markdown
## ⚠️ 背景信息处理规则（最高优先级）

**每轮对话前，Shell Hook 会自动注入以下背景信息：**

| 注入类型 | 内容示例 | 处理规则 |
|---------|---------|---------|
| **时间感知** | 当前时间、星期、市场状态 | **完全忽略，不在回复中提及** |
| **时间锚定原则** | 数据分析前必须验证时间戳 | **自动遵循，不解释原因** |
| **搜索记忆** | 近期搜索归档 | **需要时使用，不主动提及** |
| **跨会话话题** | 最近对话的话题 | **需要时搜索，不主动提及** |
```

### Memory 更新

添加到 user profile：
```
【最高优先级】背景信息处理规则：Shell Hook 注入的时间感知、时间锚定、搜索记忆、跨会话话题都是背景信息，必须完全忽略，不在回复中提及、引用或解释。只回应用户的实际问题。
```

## 验证方法

1. **自我检查**：回复中是否提及了"Shell Hook"、"背景信息"、"注入"？
2. **用户反馈**：用户是否再次纠正"不要提及时间感知"？
3. **行为验证**：是否自动遵守时间段行为准则，但不解释原因？
