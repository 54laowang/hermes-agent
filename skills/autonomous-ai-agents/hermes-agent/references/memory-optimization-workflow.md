# Memory优化工作流 - 实战经验总结

## 背景

Hermes Memory系统有5000字符限制，长期使用后会累积大量重复、过时、冗余内容，影响效率和准确性。

## 优化原则

### 1. 分层存储原则
- **Memory（热知识）**：只保留要点和触发词
- **fact_store（冷知识）**：存储详细流程和完整文档
- **Skill（程序性知识）**：存储工作流程和最佳实践

### 2. 去重优先级
1. **完全重复**：直接删除
2. **要点重复**：保留简洁版，删除详细版
3. **功能重复**：合并为一条

### 3. 过时内容识别
- 项目进度已完成（如"正在XX"）
- 技术细节已变化（如API已失效）
- 临时状态已过期（如"今天XX"）

## 优化流程

### 步骤1：诊断Memory状态
```bash
# 查看Memory使用率
# 通过memory工具的返回值查看usage字段
```

### 步骤2：分类现有条目
- **核心偏好**：用户的工作习惯、数据准确性要求、偏好格式
- **工作流程**：自动化任务、监控任务、推送任务
- **系统架构**：配置路径、关键文件位置
- **Skill触发词**：简化的功能入口描述

### 步骤3：识别优化点
- 重复内容（同一规则多处描述）
- 过时内容（项目已完成、状态已变化）
- 技术细节（API参数、代码片段应放Skill）
- 进度记录（应放fact_store或删除）

### 步骤4：执行优化
```python
# 删除过时/重复条目
memory(action="remove", old_text="...", target="memory")

# 添加精简版核心内容
memory(action="add", content="精简版内容", target="memory")

# 替换冗长版为简洁版
memory(action="replace", old_text="详细版...", content="简洁版...", target="memory")
```

### 步骤5：验证优化效果
- 检查使用率是否下降
- 确认核心信息未丢失
- 验证fact_store/Skill是否有详细版本备份

## 实战案例

### 案例1：时间锚定原则优化

**问题**：
- Memory中有4条相关条目（简要版+详细版+A股版+美股版）
- 详细版占900+ chars（18% Memory）
- 内容在fact_store#43已有完整版

**优化**：
```
删除：3条重复条目
保留：1条精简版（134 chars）
引用：指向fact_store#43获取详细流程

节省：800+ chars (16%)
```

**优化后内容**：
```
时间锚定宪法（最高优先级）：任何数据分析前必须先建立时间锚点。
A股：验证时间戳禁止用过时数据。
美股：强制加载us-stock-data-acquisition-sop Skill并完成6项检查。
详细流程见fact_store#43
```

### 案例2：项目进度清理

**问题**：
- 多条"正在XX"、"已完成XX"进度记录
- 项目已结束，进度信息无价值

**优化**：
```
删除：所有进度记录
保留：最终成果描述（如果有长期价值）

节省：~300 chars
```

### 案例3：技术细节简化

**问题**：
- Smart Router技术实现细节占200+ chars
- 网格交易API参数占150+ chars
- 这些细节在Skill中已有完整文档

**优化**：
```
删除：技术实现细节
保留：功能描述 + 效果数据
示例：Smart Skill Router已自动生效，Token节省60-70%，响应0.07ms

节省：~300 chars
```

## 优化效果统计

| 优化轮次 | 优化前 | 优化后 | 节省空间 | 优化类型 |

| 第1轮 | 80% (3,988) | 35% (1,786) | 2,202 chars | 删除重复/过时 |
| 第2轮 | 35% (1,786) | 14% (743) | 1,043 chars | 合并时间锚定条目 |

## 维护建议

### 日常维护
- 每周检查Memory使用率
- 发现超过60%时启动优化
- 及时删除项目完成后的进度记录

### 定期清理
- 每月审查重复内容
- 合并功能相似的条目
- 将技术细节迁移到Skill

### 新增条目原则
- 先检查是否已有类似内容
- 优先添加到fact_store而非Memory
- 保持简洁（单条不超过100 chars）

## 工具支持

### 诊断工具
```bash
# 查看Memory内容
memory(action="add", content="test", target="memory")  # 会返回所有条目

# 查看fact_store条目
fact_store(action="list")
```

### 清理工具
```bash
# 删除单条
memory(action="remove", old_text="...", target="memory")

# 替换
memory(action="replace", old_text="...", content="...", target="memory")
```

## 注意事项

1. **备份原则**：优化前确认重要内容在fact_store/Skill中有备份
2. **渐进优化**：分批删除，避免一次性大改
3. **验证可用性**：优化后测试核心功能是否正常触发
4. **用户确认**：大规模优化前与用户确认优化方案
