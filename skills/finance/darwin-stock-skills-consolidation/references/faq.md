## 数据获取相关

### Q1: 为什么有时候股价数据会延迟？

**A**: 数据延迟可能有以下原因：

1. **数据源限流**: 免费数据源有请求频率限制
2. **网络延迟**: 网络不稳定导致响应慢
3. **非交易时段**: 盘前/盘后数据更新较慢

**解决方案**:
- 系统会自动切换到备用数据源
- 使用智能缓存减少重复请求
- 输出中会标注数据时间戳

---

### Q2: 如何判断获取的数据是否准确？

**A**: 系统有三重验证机制：

1. **时间戳验证**: 数据时间戳<5分钟
2. **多源交叉验证**: 价格差异<0.1%
3. **数据源优先级**: 优先使用 P0 级数据源（财联社、东方财富）

**用户可检查**:
- 输出中的数据来源标注
- 数据时间戳
- 可信度评级（★数量）

---

### Q3: 美股和A股的时区如何换算？

**A**: 系统自动处理时区换算：

| 北京时间 | 美东时间（夏令时） | 美东时间（冬令时） |
|---------|------------------|------------------|
| 09:00 | 前日 21:00 | 前日 20:00 |
| 21:00 | 当日 09:00 | 当日 08:00 |
| 22:00 | 当日 10:00 | 当日 09:00 |

**夏令时判断**:
- 夏令时：3月第二个周日 → 11月第一个周日
- 冬令时：11月第一个周日 → 3月第二个周日

**用户无需手动计算**，系统会自动换算。

## 分析研判相关

### Q6: 技术分析包含哪些指标？

**A**: stock-analysis-framework 的技术分析模块包含：

| 类别 | 指标 | 用途 |
|------|------|------|
| **趋势指标** | MA、EMA、MACD | 判断趋势方向 |
| **震荡指标** | RSI、KDJ、布林带 | 判断超买超卖 |
| **成交量指标** | 成交量、换手率、量价关系 | 判断资金动向 |
| **支撑压力** | 关键价位、前高前低 | 判断买卖点位 |

---

### Q7: ST股票如何分析？

**A**: ST专项分析包含以下维度：

1. **ST类型判断**: ST、*ST、SST
2. **亏损原因拆解**: 主营业务、资产减值、投资损失
3. **报表真实性判断**: 异常利润、关联交易
4. **借壳概率评估**: 壳价值、重组预期、监管风险
5. **投资建议**: 观望/投机/回避

**示例**:
```
用户：分析一下ST某某的借壳概率
系统：
ST某某（000XXX）
ST类型：*ST（退市风险）
亏损原因：主营业务持续亏损，资产减值2.3亿
借壳概率：中等（40-60%）
风险提示：市值较大（50亿），壳价值较低
投资建议：谨慎观望
```

### Q9: 地缘政治分析包含哪些内容？

**A**: 地缘政治分析框架：

1. **事件分类**: 战争/制裁/选举/政策
2. **影响资产**: 原油、黄金、股市、汇率
3. **三情景预测**: 乐观/基准/悲观
4. **历史类比**: 类似事件的历史影响
5. **投资策略**: 对冲/投机/回避

**示例**:
```
用户：中东冲突对原油和黄金有什么影响？
系统：
【事件】中东冲突升级
【影响资产】原油、黄金
【三情景预测】
- 乐观（冲突缓和）：原油-10%，黄金-5%
- 基准（冲突持续）：原油+15%，黄金+8%
- 悲观（冲突扩大）：原油+40%，黄金+20%
【历史类比】2020年美伊冲突
【投资策略】
- 原油：短期做多，中期观望
- 黄金：避险需求，配置10%
```

### Q10: 如何评估投资风险？

**A**: 风险评估包含：

| 风险类型 | 评估指标 | 应对策略 |
|---------|---------|---------|
| **市场风险** | 波动率、回撤 | 分散投资、止损 |
| **流动性风险** | 成交量、换手率 | 避免小盘股 |
| **信用风险** | 财务健康度 | 避开财务异常公司 |
| **操作风险** | 交易频率、滑点 | 减少频繁交易 |

## 交易执行相关

### Q11: 网格交易如何配置？

**A**: 网格交易配置示例：

```json
{
  "symbol": "513130",
  "name": "恒生科技ETF",
  "base_price": 0.625,
  "grid_spacing": 0.025,
  "grid_spacing_pct": 0.04,
  "upper_grids": 6,
  "lower_grids": 8,
  "amount_per_grid": 715,
  "initial_position": 3,
  "total_capital": 10000
}
```

**参数说明**:
- `base_price`: 网格基准价（当前价格）
- `grid_spacing`: 网格间距（元）
- `grid_spacing_pct`: 网格间距百分比
- `upper_grids`: 上方卖出格数
- `lower_grids`: 下方买入格数
- `amount_per_grid`: 每格金额

---

### Q12: 网格交易如何避免频繁触发？

**A**: 系统有三重保护机制：

1. **档位冷却期**: 同一档位触发后，冷却 5 分钟
2. **网格间距控制**: 间距建议≥2%（避免频繁触发）
3. **交易时段限制**: 仅在交易时段执行

**配置示例**:
```json
{
  "cooldown_minutes": 5,
  "min_grid_spacing_pct": 0.02
}
```

---

### Q13: 网格交易在什么情况下会暂停？

**A**: 自动暂停场景：

| 触发条件 | 暂停时长 | 恢复条件 |
|---------|---------|---------|
| **异常波动预警** | 30分钟 | 市场恢复正常 |
| **数据源故障** | 故障期间 | 数据源恢复 |
| **持仓/资金不足** | 持续暂停 | 人工调整配置 |
| **非交易时段** | 休市期间 | 下一交易时段 |

### Q15: 网格交易有哪些风险？

**A**: 主要风险：

| 风险类型 | 说明 | 应对策略 |
|---------|------|---------|
| **单边上涨** | 踏空（卖飞） | 预留底仓 |
| **单边下跌** | 满仓套牢 | 设置最大持仓上限 |
| **震荡不足** | 无交易机会 | 调小网格间距 |
| **市场熔断** | 无法交易 | 自动暂停 |

## 监控预警相关

### Q16: 如何配置定时简报？

**A**: 配置示例：

```bash
# 早盘简报（11:35）
hermes cron create "35 11 * * 1-5" \
  --prompt "生成早盘简报" \
  --target weixin

# 收盘简报（15:05）
hermes cron create "5 15 * * 1-5" \
  --prompt "生成收盘简报" \
  --target weixin

# 晚间复盘（20:00）
hermes cron create "0 20 * * 1-5" \
  --prompt "生成晚间复盘简报" \
  --target weixin
```

### Q17: 简报包含哪些内容？

**A**: 简报内容：

| 类型 | 时间 | 内容 |
|------|------|------|
| **早盘简报** | 11:35 | 上午资金态度、下午看点、板块轮动 |
| **收盘简报** | 15:05 | 完整复盘、明日看点、风险提示 |
| **休市简报** | 节假日 | 节后关注点、复市倒计时、外围市场回顾 |

**核心指标**:
- 指数走势（上证/深证/创业板）
- 北向资金流向
- 两市成交额
- 涨跌停家数
- 连板股统计
- 人气热度票

### Q20: 简报风格是什么？

**A**: 采用"爱在冰川"分析风格：

**✅ 必须说**:
- 资金在什么位置做了什么选择
- 是主升加速还是高位分歧
- 情绪在周期的哪个位置
- 当前位置性价比如何

**❌ 禁止说**:
- 利好刺激上涨
- 利空导致下跌
- 震荡整理
- 市场情绪分化

## 协同使用相关

### Q21: 四大 Skills 如何协同工作？

**A**: 协同架构：

```
[监控预警层] market-intelligence-system
    ↓ 预警信号
[交易执行层] grid-trading-system
    ↓ 交易信号
[分析研判层] stock-analysis-framework
    ↓ 分析结果
[数据获取层] stock-data-acquisition
```

**典型流程**:
1. 用户提问 → 自动触发监控预警层
2. 监控预警层 → 请求数据获取层
3. 数据获取层 → 返回数据
4. 监控预警层 → 调用分析研判层
5. 分析研判层 → 返回分析结果
6. 监控预警层 → 推送给用户

---
