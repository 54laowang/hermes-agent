## 六大核心模块

### 模块一：时间锚定与交易日判断（检查点 5 个）

**核心功能**:
- 北京时间 ↔ 美东时间换算
- A股/美股市场状态判断
- A股交易日判断（节假日 + 调休）

**关键设计**:
- 时间锚定宪法（最高优先级）
- 三层判断逻辑：周末 → 节假日 → 动态交易日历
- 硬编码节假日备份（2026年）

### 模块二：多数据源管理（检查点 8 个）

**数据源优先级**:
- P0: Trading Economics, Yahoo Finance, Vibe-Trading MCP
- P1: 新浪财经, 腾讯财经, 东方财富
- P2: AkShare, 同花顺, Tushare
- P3: 硬编码节假日, 过期缓存

**关键机制**:
- 自动切换机制（P0 → P1 → P2 → P3）
- 熔断保护（连续失败 3 次触发）
- 超时控制（3-10s 按级别）

### 模块三：智能缓存系统（检查点 4 个）

**缓存策略**:
- 盘中缓存：5 分钟
- 盘后缓存：24 小时
- 休市缓存：24 小时

**关键功能**:
- 时效感知缓存
- 缓存预热（开盘前 5 分钟）
- 自动清理（7 天过期）

### 模块四：自选股管理（检查点 6 个）

**核心功能**:
- 添加/删除/查看自选股
- 实时行情获取
- 行情摘要生成

**关键设计**:
- 代码格式验证
- 股票有效性检查
- 重复检查
- 名称匹配验证

### 模块五：异常处理与降级（检查点 4 个）

**异常分类**:
- 网络异常：超时、连接失败
- 数据异常：格式错误、数据缺失
- API 异常：HTTP 4xx/5xx、Token 过期、频率限制

**降级策略**:
- 策略 A：过期缓存兜底 + 明确警告
- 策略 B：明确错误提示 + 建议操作

### 模块六：统计与监控（检查点 3 个）

**统计指标**:
- 缓存命中率（目标 ≥ 85%）
- 平均响应时间（目标 ≤ 0.5s）
- API 调用成功率（目标 ≥ 95%）

**监控告警**:
- 缓存命中率 < 70%
- 平均响应时间 > 2s
- 单数据源失败率 > 50%
