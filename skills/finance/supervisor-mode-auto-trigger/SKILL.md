# 监察者模式自动触发系统

> **核心理念**：将复杂的检查流程封装到自动化Hook中，让用户零操作即可享受高质量的监督保障。

---

## 系统架构

### 0. v2.0 方案A架构

**核心设计**：所有会话注入核心规则 + 财经关键词触发详细提示。

```
┌─────────────────────────────────────────────────────────────┐
│                     用户发送消息                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                   supervisor-precheck.py                     │
│                                                               │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  财经关键词检测？                                       │  │
│  │  □ 是 → 注入完整监察者提示（详细版）                   │  │
│  │  □ 否 → 注入核心规则（轻量级）                         │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│              所有会话都有基础保障                              │
│  - 时间锚定原则                                              │
│  - 数据源优先级                                              │
│  - 执行清单                                                  │
└─────────────────────────────────────────────────────────────┘
```

**Token 消耗对比**：
- 方案A（当前）：~200 tokens（非财经） / ~800 tokens（财经）
- 方案B（完整版）：~800 tokens（所有会话）
- **节省**：非财经任务节省 75% tokens

### 1. Shell Hook自动触发

**触发条件**（满足任一即触发详细提示）：

#### 美股关键词
- "美股" + ("走势" / "收盘" / "行情" / "分析")
- "道琼斯" / "纳斯达克" / "标普500" / "S&P"
- "Fed" / "美联储" / "利率决议"
- "Dow" / "Nasdaq"
- "earnings" / "财报"

#### A股关键词
- "A股" + ("走势" / "收盘" / "行情" / "分析")
- "上证指数" / "深证成指" / "创业板"
- "财联社" / "东方财富"

**触发后行为**：
1. 检测市场类型（us_stock / a_stock）
2. 自动计算时间锚定（北京时间 ↔ 美东时间）
3. 判断市场状态（盘前/盘中/收盘/盘后）
4. 注入数据源优先级和执行清单
5. 将完整提示追加到系统提示中

## 检查点体系

### 检查点0：时间锚定验证（最高优先级）

```
□ 当前北京时间：__________
□ 美东时间：__________（仅美股）
□ 目标数据日期：__________
□ 搜索关键词是否包含正确日期：□ 是 □ 否
□ 数据日期是否匹配目标日期：□ 是 □ 否
```

**时区换算公式**：
- 美东时间 = 北京时间 - 12小时
- 美东收盘日期 = 北京时间日期 - 1（凌晨04:00后）

### 检查点1：数据类型验证

- 数据类型是否匹配市场状态？
- 是否在正确的时间节点获取数据？

### 检查点2：数据完整性验证

- 所有必需数据是否已获取？
- 数据字段是否完整？

### 检查点3：数据源权威性验证

- 是否优先使用P0级数据源？
- P0失败是否说明原因？
- 是否使用禁止的数据源？

### 检查点4：数据冲突检测

- 多数据源是否一致？
- 差异是否≥0.1%？
- 是否需要用户确认？

### 检查点5：用户确认机制

- 是否触及确认触发条件？
- 是否已征求用户同意？

### 检查点6：逻辑一致性验证

- 分析结论是否与数据一致？
- 是否存在逻辑矛盾？

## 数据冲突检测机制

### 冲突类型

1. **数值冲突** - 不同数据源数值差异≥0.1%
2. **时间冲突** - 时间戳差异≥5分钟
3. **来源冲突** - P0与P2级数据源不一致

### 处理流程

1. 优先采用P0级数据源
2. P0级内优先采用最新时间戳
3. 差异≥0.1%时主动告知用户
4. 展示所有数据源和差异
5. 询问用户采用哪个数据源

## 用户确认机制

### 触发条件

1. **关键决策前** - 买入/卖出建议、投资组合调整
2. **数据冲突无法自动解决** - 差异≥0.1%
3. **时间锚定不确定** - 跨日事件、时区存疑
4. **数据来源权威性存疑** - 只能获取P2级数据

### 确认方式

```
🔔 需要您的确认
━━━━━━━━━━━━━━━━━━━━
当前任务：{task_name}

数据来源：
  - {source_1}（P0级）✓
  - {source_2}（P0级）✓

数据置信度：{confidence}
时间锚定：{time_status}

⚠️ 需确认事项：
  {issue_description}
  
建议：{recommendation}
是否确认？□ 继续 □ 调整 □ 等待
```

## 常见问题

### Q1: Hook是否会增加延迟？

**A**: 不会。Hook执行时间 < 0.1秒，对用户无感知。

### Q2: 如何添加新的关键词？

**A**: 修改 `supervisor-precheck.py` 中的正则表达式模式：

```python
us_stock_patterns = [
    r'美股.*(走势|收盘|行情|分析)',
    r'(道琼斯|纳斯达克|标普500)',  # ← 添加新关键词
    # ...
]
```

### Q3: 如何禁用自动触发？

**A**: 注释掉 `config.yaml` 中的Hook配置：

```yaml
hooks:
  pre_llm_call:
  # - command: /Users/me/.hermes/hooks/supervisor-precheck.py
  #   timeout: 5
```

### Q4: 如何扩展到其他市场（港股/日股）？

**A**: 
1. 添加新市场关键词到 `should_trigger_supervisor()`
2. 添加新市场逻辑到 `generate_supervisor_prompt()`
3. 定义新市场的时区换算规则
