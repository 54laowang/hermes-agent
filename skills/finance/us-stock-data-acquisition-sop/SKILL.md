## 第二步：时区换算（关键！）

### 美东时间 = 北京时间 - 12小时

| 北京时间 | 美东时间 | 说明 |
|----------|----------|------|
| 21:30 | 09:30 | 美股开盘 |
| 04:00（次日） | 16:00 | 美股收盘 |
| 02:00（次日） | 14:00 | 美联储决议公布 |

### 关键换算公式

```
美东收盘日期 = 北京时间日期 - 1（凌晨4:00后）
```

**示例**：
- 北京时间 2026-04-30 05:30
- 美东时间 = 2026-04-29 17:30
- 美股已收盘（收盘时间美东16:00 = 北京时间次日04:00）
- **应搜索：美东4月29日收盘数据**

| 北京时间 | 美东时间 | 市场状态 | 数据类型 |
|----------|----------|----------|----------|
| 21:30前 | 9:30前 | 盘前 | 昨日收盘 + 盘前期货 |
| 21:30-04:00 | 9:30-16:00 | 盘中 | 实时行情（不推荐分析） |
| **04:00后** | **16:00后** | **收盘** | **收评数据** |
| 周末 | 周末 | 休市 | 上周五收盘数据 |

## 常见错误（必须避免）

| 错误行为 | 后果 | 正确做法 |
|----------|------|----------|
| 混淆美东/北京时间 | 数据日期错误 | 明确标注"美东时间"或"北京时间" |
| 用北京时间日期直接搜索 | 返回错误日期数据 | 先换算为美东日期 |
| 用4月27日数据代替4月29日 | 引用过时数据 | 验证数据时间戳 |
| 搜索"A股收盘"格式搜索美股 | 返回A股数据 | 使用美股专用关键词 |

## 🚫 绝对禁止行为

1. **❌ 禁止用过时数据凑数**
   - 没有正确日期数据时，宁可暂停搜索，不用其他日期代替
   - 例：用户要求4月29日数据，搜索失败时不能用4月27日数据

2. **❌ 禁止跳过时区换算**
   - 必须先完成"执行清单"的6项检查
   - 时区换算错误 = 后续所有分析无效

3. **❌ 禁止用中文关键词搜索美股**
   - 首次搜索必须用英文确认数据准确性
   - 中英文双搜索验证

4. **❌ 禁止在时间不确定时输出报告**
   - 数据时间戳未验证通过 → 不输出分析报告
   - 宁可告知用户"数据获取失败，需进一步验证"

5. **❌ 禁止在数据不完整时输出报告（新增）**
   - 关键数据缺失时，必须触发用户确认
   - 明确标注缺失项，不隐瞒问题
   - 提供替代方案（等待数据更新、使用其他数据源）
   - 例：京东分析中，美股缺少4/29确切数据时，应先询问用户是否继续

## ⚠️ 数据冲突检测机制

### 冲突类型

**1. 数值冲突**
```
示例：
  数据源A（Trading Economics）：标普500 = 7,135.95
  数据源B（Yahoo Finance）：标普500 = 7,136.02
  差异：0.07点
```

**2. 时间冲突**
```
示例：
  数据源A时间戳：2026-04-29 16:00 ET
  数据源B时间戳：2026-04-29 16:05 ET
  差异：5分钟
```

**3. 来源冲突**
```
示例：
  数据源A权威性：P0级
  数据源B权威性：P2级
  优先采用：数据源A
```

### 处理机制

**自动处理**：
1. 优先采用P0级数据源
2. P0级内优先采用最新时间戳
3. 数值差异<0.1%视为合理误差

**用户确认**：
1. 差异≥0.1%时主动告知用户
2. 展示所有数据源和差异
3. 询问用户采用哪个数据源

**冲突报告格式**：
```
⚠️ 数据冲突检测
━━━━━━━━━━━━━━━━━━━━
数据类型：收盘价
冲突类型：数值冲突

数据源A：Trading Economics（P0级）
  - 时间戳：2026-04-29 16:00 ET
  - 数值：标普500 = 7,135.95

数据源B：Yahoo Finance（P0级）
  - 时间戳：2026-04-29 16:00 ET
  - 数值：标普500 = 7,136.02

差异：0.07点（0.001%）

处理建议：差异极小，建议采用数据源A
请确认：□ 采用A □ 采用B □ 使用平均值
```

## 🔔 用户确认机制

### 触发条件（满足任一即触发）

1. **关键决策前**
   - 买入/卖出建议
   - 投资组合调整
   - 风险提示

2. **数据冲突无法自动解决**
   - 差异≥0.1%
   - 多个P0级数据源不一致

3. **时间锚定不确定**
   - 跨日事件（如美联储决议）
   - 时区换算存疑

4. **数据来源权威性存疑**
   - 只能获取P2级数据
   - 数据源无时间戳
   - 数据来源不明

### 确认方式

**标准流程**：
```
1. 展示数据来源和置信度
2. 说明不确定性原因
3. 询问用户是否继续分析
4. 提供替代方案
```

**确认模板**：
```
🔔 需要您的确认
━━━━━━━━━━━━━━━━━━━━
当前任务：美股走势分析

  - Trading Economics（P0级）✓
  - Yahoo Finance（P0级）✓
  - CNBC（P1级）✓

数据置信度：高
时间锚定：已验证（美东4月29日收盘）

⚠️ 需确认事项：
  检测到数据冲突：
  - 标普500收盘价差异：0.07点（0.001%）
  
建议继续分析，是否确认？
□ 继续分析（推荐）

```
