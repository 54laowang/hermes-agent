# 多数据源交叉验证技术

## 概述

本文档记录了从用户纠正中学习的**股价数据多源交叉验证**技术。

## 背景

**用户反馈**（2026-04-30）：
> "这个股价显示不准确！按照时间应该是收盘价格，我不是配置了有api，应该用api数据获取多个数据源交叉验证一下"

**问题分析**：
- 仅依赖web_search返回的数据不一致
- 不同数据源可能返回不同日期的数据（盘中 vs 收盘）
- 用户期望使用配置的API进行多源验证

## 解决方案

### 方法：使用execute_code进行多源验证

```python
from hermes_tools import web_search
import json

# 农尚环境（300536）示例
search_queries = [
    {"query": "农尚环境 300536 收盘 东方财富", "source": "eastmoney"},
    {"query": "农尚环境 300536 收盘 新浪财经", "source": "sina"},
    {"query": "农尚环境 300536 收盘 雪球", "source": "xueqiu"},
]

# 执行多源搜索
results = {}
for i, q in enumerate(search_queries):
    result = web_search(query=q["query"], limit=3)
    results[f"search_{i}"] = {
        "query": q["query"],
        "source": q["source"],
        "data": result.get("data", {}).get("web", [])
    }

# 输出并分析
print(json.dumps(results, indent=2, ensure_ascii=False))
```

## 验证规则

### 1. 时间戳优先原则

- ✅ 时间戳明确且匹配目标时间的数据优先
- ❌ 时间戳模糊或不匹配的数据排除

**示例**：
- 目标：2026-04-30 15:00收盘数据
- ✅ "2026-04-30 15:04:42 北京时间" - 匹配
- ❌ "2026-04-30 10:11:18" - 盘中数据，不匹配

### 2. 多数一致原则

如果多个数据源数据一致，可信度高：
- 3个数据源一致 → 可信度0.95
- 2个数据源一致 → 可信度0.85
- 仅1个数据源 → 可信度0.70

### 3. 数据源优先级

冲突时优先选择：
1. P0级（东方财富、财联社）
2. P1级（新浪、雪球、腾讯）
3. P2级（其他平台）

### 4. 明确标注来源

在分析报告中必须标注：
```
**数据来源**：雪球 ✅
**时间验证**：2026-04-30 15:04 ✅
**监察者确认**：✅ 数据源可信度符合P1级标准
```

## 常见陷阱

### 陷阱1：使用盘中数据代替收盘数据

**错误示例**：
```
搜索："农尚环境 300536 今日行情"
返回：10.07元（2026-04-30 10:11盘中）
误判：将盘中数据作为收盘数据使用
```

**正确做法**：
```
搜索："农尚环境 300536 2026年4月30日 收盘价"
验证：数据时间戳是否为15:00后
```

### 陷阱2：不同数据源数据不一致

**错误示例**：
```
数据源A：8.97元
数据源B：5.42元
行动：随意选择其中一个
```

**正确做法**：
```
对比时间戳、数据源可信度
明确标注："数据源冲突，优先使用P0级数据源"
或向用户说明冲突情况
```

### 陷阱3：忽略数据源被拦截

**错误示例**：
```
web_extract("http://quote.eastmoney.com/...")
返回：Blocked
行动：忽略，继续使用其他数据
```

**正确做法**：
```
记录："东方财富数据源被拦截"
降级：使用P1级数据源
标注："P0级数据源不可用，使用P1级数据源"
```
