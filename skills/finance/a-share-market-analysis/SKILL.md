## 一、数据获取SOP（时间锚定原则）

> **强制规则**：这是所有A股分析的前置步骤，优先级高于一切。

### 第一步：读取当前时间

系统已自动注入时间，直接读取：
```
当前时间：{system_injected_time}
```

### 第二步：判断市场状态

| 时间段 | 市场状态 | 数据类型 |
|--------|----------|----------|
| 9:30前 | 盘前 | 昨日收盘 + 隔夜外盘 |
| 9:30-11:30 | 早盘盘中 | 实时行情（不推荐分析） |
| **11:30** | **午盘收盘** | **午评数据** |
| 11:30-13:00 | 午间休盘 | 午盘数据（此时可用） |
| 13:00-15:00 | 午后盘中 | 实时行情（不推荐分析） |
| **15:00后** | **收盘** | **收评数据** |
| 周末 | 休市 | 上周五收盘数据 |

### 第三步：精确搜索关键词

**必须包含三要素：日期 + 时间节点 + 数据类型**

#### 午盘数据（11:30-13:00）
```
"A股午间收盘 {YYYY年M月D日}"
"午评 {M月D日} 沪指 创业板指"
"财联社 {M月D日} 午评"
```

#### 收盘数据（15:00后）
```
"A股收盘 {YYYY年M月D日} 沪指 深成指 创业板指"
"收评 {M月D日} 涨跌"
"财联社 {M月D日} 收评"
```

#### 盘前数据（9:30前）
```
"A股昨日收盘 {YYYY年M月D日}"
"隔夜外盘 {M月D日}"
"早盘策略 {M月D日}"
```

### 第四步：使用API获取数据并交叉验证

**⚠️ 强制规则**：必须使用API获取股价数据，禁止仅依赖web_search

#### 数据获取优先级

**方式1：使用配置的API（首选）**

**方式3：web_search作为最后手段**
- 仅在API不可用时使用
- 必须明确搜索日期和时间节点
- 必须验证返回数据的时间戳

#### 数据源可信度评分

| 数据源 | 级别 | 可信度 | 特点 |
|--------|------|--------|------|
| 东方财富 | P0 | 0.90 | 官方数据源，最权威 |
| 新浪财经 | P1 | 0.80 | 数据全面，更新及时 |
| 雪球 | P1 | 0.75 | 时间戳明确，用户活跃 |
| 腾讯财经 | P1 | 0.70 | 港股数据较好 |
| 其他平台 | P2 | 0.50 | 需谨慎验证 |

#### 交叉验证规则

1. **时间戳优先**：选择时间戳最接近目标时间的数据
2. **多数一致**：如果多个数据源一致，可信度高
3. **冲突处理**：数据源冲突时，优先P0级数据源
4. **标注来源**：在分析报告中明确标注数据来源

**P0级数据源（首选）：**
1. 财联社盘面直播：https://www.cls.cn/subject/1103
2. 东方财富网：https://stock.eastmoney.com/

**P1级数据源（备选）：**
1. 新浪财经
2. 雪球
3. 证券时报

**❌ 禁止行为：**
- 禁止仅依赖web_search而不验证时间戳
- 禁止使用过时数据凑数（如用昨日收盘代替今日午盘）
- 禁止忽略数据源冲突而不说明

### 第五步：验证数据时间戳

**强制验证项：**
- [ ] 返回数据的日期是否 = 今天？
- [ ] 返回数据的时间节点是否 = 当前市场状态？

**如果验证失败：**
1. 立即停止分析
2. 重新搜索（换关键词/换数据源）
3. 禁止用错误时间的数据凑数

## 二、ST公司财务分析框架

### 适用场景

- *ST公司退市风险评估
- ST公司借壳/重整预期研判
- 困境反转投资机会筛选

### 分析维度

#### 1. 亏损原因拆解

| 亏损类型 | 特征 | 风险等级 |
|----------|------|----------|
| 周期性亏损 | 行业低谷、价格下跌 | 低（可反转） |
| 经营性亏损 | 主营业务萎缩 | 中（需转型） |
| 财务性亏损 | 债务沉重、利息高 | 高（需减债） |
| 突发性亏损 | 计提减值、诉讼赔偿 | 需评估持续性 |

#### 2. 报表真实性判断

- 应收账款周转率异常 → 收入质量存疑
- 存货周转率异常 → 滞销风险
- 经营现金流与净利润背离 → 盈利质量差
- 关联交易占比过高 → 利润转移风险

#### 3. 借壳预期评估

**正向信号：**
- 市值小（<30亿）
- 债务相对干净
- 大股东有优质资产
- 注册地/主营业务稀缺

**负向信号：**
- 诉讼缠身
- 大额担保
- 资产被冻结
- 监管立案调查

#### 4. 重整预期评估

**重整成功概率：**
- 地方政府支持力度
- 产业投资人背景
- 债权人结构（银行占比高则更易谈判）
- 核心资产价值

## 三、市场状态判断流程

### 盘口分析

```
市场状态 = f(涨跌比, 成交量, 板块轮动, 北向资金)
```

#### 关键指标

| 指标 | 计算方式 | 意义 |
|------|----------|------|
| 涨跌比 | 上涨家数/下跌家数 | 市场情绪 |
| 量比 | 当日成交量/5日均量 | 放量/缩量 |
| 板块热度 | 涨停板块数/总板块数 | 资金活跃度 |
| 北向净流入 | 当日净买入额 | 外资态度 |

#### 状态判断

| 涨跌比 | 量比 | 判断 |
|--------|------|------|
| >1.5 | >1.2 | 强势上涨 |
| >1.5 | <0.8 | 缺量上涨（谨慎） |
| <0.67 | >1.2 | 放量下跌（恐慌） |
| <0.67 | <0.8 | 缩量下跌（筑底） |

## 四、常见错误与纠正

| 错误行为 | 后果 | 正确做法 |
|----------|------|----------|
| 用11:05的盘中快讯代替11:30午盘数据 | 数据不准确 | 等11:30后搜索午评 |
| 用昨日收盘数据代替今日午盘 | 时间错误 | 明确搜索"今日午评" |
| 搜索"A股今日行情" | 返回模糊结果 | 明确搜索"午间收盘+日期" |
| 不验证数据时间戳 | 引用过时数据 | 检查数据发布时间 |

**关联技能**：`stock-price-cache`

**使用场景**：

- 批量查询多只股票时
- 避免重复API调用被限流

**核心优势**：
- 响应速度：300ms → 5ms（60倍提升）
- 智能时效：根据交易时间自动判断缓存有效期
- 多源冗余：主数据源失败自动切换备用源

**快速使用**：
```bash

python3 ~/.hermes/skills/stock-price-cache/scripts/stock_query.py 300536

# 查看缓存状态
python3 ~/.hermes/skills/stock-price-cache/scripts/stock_query.py --status

# 强制更新
python3 ~/.hermes/skills/stock-price-cache/scripts/stock_query.py --force 300536
```

**在代码中使用**：
```python
import sys
sys.path.insert(0, '~/.hermes/skills/stock-price-cache/scripts')

# 获取股价（自动缓存）

```
