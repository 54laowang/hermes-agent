## 🧬 达尔文进化说明

本 Skill 由 **达尔文进化方法论** 指导整合而成，遵循四大原则：

1. **自然选择**: 保留核心财务能力（核算/建模/规划/风控），淘汰冗余流程

## 模块二：财务建模与分析

### 核心原则

**先陈述假设，再给出结论**：每个模型都基于假设，如果利益相关方看不到假设，他们就无法质疑。

**必须构建场景分析**：永远不要呈现单点预测，提供基准、乐观和悲观场景。

### 财务建模

#### 三表联动模型
```
利润表 → 资产负债表 → 现金流量表

关键链接：
  • 净利润 → 留存收益
  • 折旧/摊销 → 固定资产/无形资产
  • 营运资金变动 → 经营性现金流
```

**检查点 2.1**: 三表勾稽关系验证
**检查点 2.2**: 模型无循环引用

#### DCF 估值
```
企业价值 = Σ(未来自由现金流 / (1+WACC)^t) + 终值

关键假设：
  • WACC 计算（无风险利率、β、市场风险溢价）
  • 终值方法（永续增长法、退出倍数法）
  • 敏感性分析（WACC、增长率）
```

**检查点 2.3**: WACC 计算合理
**检查点 2.4**: 终值假设明确
**检查点 2.5**: 敏感性分析完成

#### LBO 模型
```
杠杆收购核心：
  • 债务明细（优先债、次级债）
  • 回报分析（IRR、MOIC）
  • 信用指标（杠杆倍数、利息覆盖率）
```

**检查点 2.6**: 债务结构合理
**检查点 2.7**: IRR 计算准确

### 分析框架

#### 差异分析
```
差异分解：
  • 价格差异 = (实际价格 - 预算价格) × 实际数量
  • 数量差异 = (实际数量 - 预算数量) × 预算价格
```

**检查点 2.8**: 差异根因已识别

#### 单位经济
```
SaaS 核心指标：
  • CAC（获客成本）
  • LTV（客户终身价值）
  • LTV/CAC > 3x（健康值）
  • 回本周期 < 12 个月
```

**检查点 2.9**: 单位经济指标已计算

#### 场景规划
```
三场景建模：
  • 基准场景（概率 40-60%）
  • 乐观场景（概率 20-30%）
  • 悲观场景（概率 20-30%）
```

**检查点 2.10**: 三场景已构建
**检查点 2.11**: 概率分配合理

## 模块四：创业公司财务预测

### 核心原则

**现金流安全底线**：任何时候 Runway 不得低于 6 个月。

**预测诚信**：所有预测必须标注关键假设和数据来源。

### 收入预测

#### SaaS 收入建模
```
MRR 计算：
  • 新增 MRR = 新客户数 × 平均客单价
  • 增购 MRR = 升级客户数 × 增购金额
  • 流失 MRR = 流失客户数 × 平均客单价
  • 净增 MRR = 新增 + 增购 - 流失

ARR = MRR × 12
NDR = (期初 ARR + 增购 - 流失) / 期初 ARR
```

**检查点 4.1**: MRR/ARR 已计算
**检查点 4.2**: NDR 已计算（目标 > 110%）

### 现金流管理

#### 13 周滚动现金流预测
```
周度预测：
  • 期初现金
  • + 经营性现金流入
  • - 经营性现金流出
  • - 资本性支出
  • = 期末现金
  
Runway = 当前现金 / 月度 Net Burn
```

**检查点 4.3**: 13 周滚动预测已完成
**检查点 4.4**: Runway 已计算
**检查点 4.5**: Runway 预警机制已设置

```
必备指标：
  • ARR 及增长率
```

```
  • 凌晨 0:00-6:00 大额交易 → 延迟结算

  • 距离 > 500km 且无出行记录 → 拦截+人脸验证
```

**检查点 5.1**: 实时规则已配置
**检查点 5.2**: 规则阈值已验证
