# 小说章节 AI 痕迹去人性化修订流程（中文）

## 适用场景
- 用户要求原创小说章节创作
- 每章创作完成后需要去除AI生成痕迹，让文笔更自然流畅
- 适合长篇连载小说，逐章创作逐章修订

## 修订规范（humanizer-zh 标准）

### 核心原则
1. **删除AI过度修饰**：去掉冗长排比、过度铺陈、AI式自问自答
2. **调整句子节奏**：合并零散短句，拆分过长句子，让读起来顺口
3. **去除AI特征表达**：删掉"在某个阳光明媚的早晨"这种模板化开头，删掉不必要的形容词堆砌
4. **保留悬念留白**：重要转折不要说破，给读者想象空间
5. **保持叙事自然**：符合人物说话习惯，符合情节发展逻辑

### 具体操作步骤

1. **草稿创作**：根据大纲写出章节内容
2. **第一轮修订**：
   - 删掉所有"他心想"、"他不禁想到"这种冗余表述
   - 合并零散短句，让段落更连贯
   - 去掉重复形容词，保留最有力量的那个
3. **第二轮修订**：
   - 检查逻辑链是否完整，关键人物出场是否铺垫
   - 补全缺失的过渡情节，解决突兀问题
   - 调整对话，让每个人物说话符合身份性格
4. **质量评分**：按50分制评分，45分以上合格
   - 45-50：逻辑完整，语言自然，悬念到位
   - 40-44：基本合格，局部需要微调
   - <40：大结构有问题，需要重写
5. **保存文件**：所有章节合并保存在同一个markdown文件中

## 常见问题处理

| 问题 | 解决方法 |
|------|----------|
| 人物出场突兀 | 提前1-2个段落加铺垫，交代人物从哪里来，为什么在这里 |
| AI排比过多 | 删掉多余句子，保留最核心的一句 |
| 对话不自然 | 缩短长句，改成口语化表达 |
| 节奏拖沓 | 删掉无关风景描写，直接进入情节 |

## 完整示例流程

1. 用户要求继续创作 → 写出草稿 → 自动修订 → 用户指出问题（如突兀）→ 补充情节 → 再次修订 → 保存文件
2. 每次修订后都把完整修订版单独展示给用户，同时更新文件
3. 修订完成后询问是否继续创作下一章节
4. 用户要求长回复分段发送时，遵循用户要求分段输出，避免一次性发送被截断

## 质量检查清单

- [ ] 人物出场逻辑通顺吗？
- [ ] 有没有AI式排比和过度修饰？
- [ ] 句子节奏自然吗？有没有一口气读不通的长句？
- [ ] 悬念保留了吗？结尾能吸引读者看下一章吗？
- [ ] 符合核心设定吗？人物性格一致吗？
