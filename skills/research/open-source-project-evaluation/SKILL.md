## 模块一：项目状态快速判断

### 触发场景

当用户询问：
- "如何安装使用这个项目"
- "查看下这个 GitHub 项目"
- "这个开源项目怎么样"
- "这个固件能刷到我的设备上吗"
- "刷了之后能实现什么功能"

### 快速检查清单（30秒内完成）

```bash
# 克隆仓库检查文件结构
git clone <repo_url> /tmp/<repo_name>
ls -la /tmp/<repo_name>/

# 检查 README 完整度
wc -l /tmp/<repo_name>/README.md  # <50行 = 可能不完整
cat /tmp/<repo_name>/README.md

# 检查项目活跃度
git log --oneline -n 10  # 最近10次提交
git log --since="6 months ago" --oneline | wc -l  # 近半年提交数
```

### 判断标准

| 状态判断 | README 行数 | 代码文件 | 依赖说明 | 可立即使用 |
|---------|-----------|---------|---------|----------|
| **完整可用** | >200行 | ✅ 有 | ✅ 完整 | ✅ 是 |
| **需要评估** | 50-200行 | ✅ 有 | 🟡 部分 | 🟡 需进一步分析 |
| **学术项目** | <50行 | ❌ 无 | ❌ 无 | ❌ 否（代码未开源） |
| **已废弃** | 任意 | ✅ 有 | ❌ 过时 | ❌ 否（依赖失效） |

#### 【检查点 1】项目状态验证

**检查内容**:
- [ ] README 文件已读取
- [ ] 代码目录结构已检查
- [ ] 最近提交记录已查看
- [ ] 项目状态已判断（完整可用/需要评估/学术项目/已废弃）

**验证方法**:
```python
def check_project_status(repo_path):
    readme_lines = count_lines(f"{repo_path}/README.md")
    has_code = check_code_files(repo_path)
    recent_commits = get_recent_commits(repo_path, months=6)
    
    if readme_lines > 200 and has_code and recent_commits > 10:
        return "完整可用"
    elif readme_lines < 50 and not has_code:
        return "学术项目"
    # ... 其他判断逻辑
```

**失败处理**: 项目状态不确定时，按"需要评估"处理，继续后续模块。

## 模块二：通用可用性评估

### 安装难度评估

| 难度等级 | 判断标准 | 预计时间 |
|---------|---------|---------|
| 🟢 简单 | 有 Docker/一键脚本 | <10分钟 |
| 🟡 中等 | 需手动安装依赖，有详细文档 | 10-60分钟 |
| 🟠 较难 | 依赖复杂，文档不完整 | 1-3小时 |
| 🔴 困难 | 需要编译/特殊环境，无文档 | >3小时 |

### 依赖复杂度分析

```bash
# Python 项目
cat requirements.txt | wc -l  # 依赖包数量
pip install -r requirements.txt --dry-run  # 预检查

# Node.js 项目
cat package.json | jq '.dependencies | length'

# IoT/嵌入式项目
grep -r "idf.py menuconfig"  # 是否需要配置
cat partitions_*.csv  # Flash 容量要求
```

#### 【检查点 2】安装难度验证

**检查内容**:
- [ ] 安装方式已确认（Docker/源码编译/包管理器）
- [ ] 依赖数量已统计
- [ ] 潜在冲突已识别
- [ ] 安装难度已评级

### 文档质量评分

| 维度 | 评分标准 | 分值 |
|------|---------|------|
| **安装指南** | 是否有详细的安装步骤 | 0-25分 |
| **使用示例** | 是否有可运行的代码示例 | 0-25分 |
| **API 文档** | 是否有完整的 API 参考 | 0-20分 |
| **故障排查** | 是否有常见问题解答 | 0-15分 |
| **更新日志** | 是否有 CHANGELOG | 0-15分 |

**评分等级**:
- 80-100分：优秀
- 60-80分：良好
- 40-60分：一般
- <40分：较差

## 模块五：风险与注意事项

### 技术风险

- ⚠️ **驱动不兼容**：不同厂商硬件引脚不同
- ⚠️ **性能瓶颈**：MCU 算力有限，复杂任务要云端
- ⚠️ **稳定性**：开源项目可能有未修复 bug
- ⚠️ **依赖失效**：第三方依赖可能停止维护

### 使用风险

- ⚠️ **保修失效**：刷第三方固件可能失去保修
- ⚠️ **变砖风险**：烧录不当可能导致设备无法启动
- ⚠️ **功能缺失**：可能丢失原固件的某些专有功能
- ⚠️ **安全漏洞**：开源固件可能存在未发现的安全问题
