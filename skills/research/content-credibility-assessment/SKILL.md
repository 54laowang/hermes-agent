# 内容可信度评估框架

## 核心目标

提供系统化的内容可信度评估，帮助用户识别虚假、夸大或误导性信息，做出理性判断。

## 评估维度

### 1️⃣ 来源可信度（Source Credibility）

**评级标准**：
- ★★★★★ 官方权威来源（政府、顶级期刊、知名企业官网）
- ★★★★☆ 主流媒体、知名专家、高信誉平台
- ★★★☆☆ 中等信誉媒体、有一定影响力的博主
- ★★☆☆☆ 自媒体、营销号、个人博客
- ★☆☆☆☆ 匿名来源、三无账号、不明平台

**检查项**：
- [ ] 作者/发布者身份是否可追溯？
- [ ] 平台是否有审核机制？
- [ ] 是否有历史发布记录？
- [ ] 是否有利益冲突？（软文、推广）

### 2️⃣ 内容合理性（Content Plausibility）

**合理性检查**：
- [ ] **逻辑自洽性**：内部逻辑是否一致？
- [ ] **常识符合度**：是否违背基本常识？
- [ ] **数据支撑**：关键论断是否有数据/证据？
- [ ] **来源引用**：是否引用可信来源？
- [ ] **时间戳明确**：数据/案例是否有明确时间？

**红旗警示**（Red Flags）：
- ⚠️ 标题党、情绪化表达
- ⚠️ 绝对化断言（"一定"、"必定"、"100%"）
- ⚠️ 缺少具体数据/来源
- ⚠️ 时间信息模糊或缺失
- ⚠️ 回避反面证据/风险
- ⚠️ 制造紧迫感/焦虑感

### 3️⃣ 证据质量（Evidence Quality）

**证据分级**：
- **P0（最高优先级）**：官方公告、财报、交易所数据
- **P1**：主流财经媒体、Wind/Bloomberg、知名分析师
- **P2**：行业媒体、专业人士分析
- **P3**：自媒体、论坛、个人观点

**交叉验证**：
```
至少需要 3 个独立数据源
时间戳一致性检查
数据性质确认（涨/跌/平）
数值逻辑合理性检验
```

### 4️⃣ 利益动机（Incentive Analysis）

**利益相关方识别**：
- [ ] 发布者是否直接获利？
- [ ] 是否存在软文/广告嫌疑？
- [ ] 是否在推销产品/服务？
- [ ] 是否在制造焦虑收割流量？

**典型套路**：
- 副业焦虑营销："轻松赚钱"、"睡后收入"、"普通人也能复制"
- 知识付费套路："限时优惠"、"错过后悔"、"独家揭秘"
- 投资骗局特征："保本保息"、"稳赚不赔"、"内幕消息"

### 5️⃣ 实际可行性（Practical Feasibility）

**可行性评估**：
- [ ] 声称的时间/成本是否合理？
- [ ] 是否隐藏关键门槛/难度？
- [ ] 实际投入产出比如何？
- [ ] 是否有成功案例？样本量？

**幸存者偏差检测**：
- 是否只展示成功案例？
- 失败率/成功率是多少？
- 是否经过第三方验证？

## 特殊场景评估模板

### GitHub 项目评估

**评估要点**：
- 项目成熟度（提交数、Star 数、Fork 数）
- 社区活跃度（Issue 响应、PR 合并频率）
- 文档质量（README 完整度、示例代码）
- 实际使用案例
- 与现有方案的对比

**评估模板**：
```markdown
## 基本信息
- Star：X 个
- Fork：Y 个
- 提交：Z 次
- 更新：最后更新时间

## 优点
1. [优点1]
2. [优点2]

## 问题
1. [问题1]
2. [问题2]

## 对比现有方案
| 维度 | 本项目 | 现有方案 |
|------|--------|---------|
| [维度1] | [对比] | [对比] |

## 最终评分：★★★☆☆
```

### WeChat 公众号文章评估

**特别关注**：
- 是否为营销号？（标题党、情绪化、推广多）
- 数据来源是否明确？
- 案例是否可验证？
- 是否回避风险/成本？

## 实战案例

### 案例 1：副业营销文章

**输入**："30秒赚127美元：普通人也能复制的AI副业玩法"

**评估输出**：
```markdown
## 🔍 可信度评估：★★☆☆☆（偏低）

### ✅ 合理的部分
1. POD 商业模式真实存在 - Printify、Redbubble 等平台确实运营中
2. 技术门槛低 - AI 绘图工具降低了设计门槛

### ⚠️ 严重夸大的部分
1. "30秒赚127美元"极具误导性 - 30秒只是生成图案时间，不包括运营推广
2. 回避核心问题 - 流量从哪来？竞争激烈程度？
3. 缺少关键信息 - 作者真实身份、实际操作经验

### 🚩 典型营销套路
- 标题党："30秒赚127美元"
- 制造暴富幻想："躺着等钱"、"睡后收入"
- 低门槛诱惑："普通人也能复制"
- 回避风险：完全不提流量、竞争、版权

### 💡 建议
- 不要被标题忽悠
- POD 模式可以做，但要当成正经副业投入时间
- 先小规模测试
```

## 注意事项

1. **保持客观**：不要预设立场，基于证据评估
2. **避免绝对化**：用"可能"、"倾向于"等表述
3. **提供替代视角**：即使内容可疑，也指出合理部分
4. **给出可执行建议**：不仅说"不可信"，还要说"应该怎么做"
5. **避免过度怀疑**：不是所有内容都是骗局，理性判断

## ✅ Done When 完成判据

> **核心思想**：从"我猜我做完了"变成"我能确认我做完了"

### 四大支柱

| 支柱 | 说明 | 本 Skill 对应 |
|------|------|--------------|
| **Goal** | 任务目标 | 完成内容可信度评估并给出明确建议 |
| **Context** | 上下文来源 | 用户分享的内容链接/文本 |
| **Constraints** | 约束条件 | 五维评估框架、证据分级、交叉验证 |
| **Done When** | 完成判据 | 下方必检项 |

### 必检项（全部满足才算完成）

#### 【任务：内容可信度评估】

- [ ] **来源可信度已评级**
  - 作者/发布者身份已确认
  - 平台信誉已判断
  - 利益冲突已识别
  - 来源星级已评定（★☆☆☆☆ ~ ★★★★★）
  - **验证方法**：输出包含「来源可信度」评分

- [ ] **内容合理性已检查**
  - 逻辑自洽性已验证
  - 常识符合度已判断
  - 数据支撑已确认
  - 红旗警示已识别（如有）
  - **验证方法**：输出包含「内容合理性」评分 + 红旗列表（如有）

- [ ] **证据质量已评估**
  - 证据分级已完成（P0-P3）
  - 交叉验证已完成（至少 3 个独立源）
  - 时间戳一致性已检查
  - **验证方法**：输出包含「证据质量」评分 + 数据源列表

- [ ] **利益动机已分析**
  - 发布者获利方式已识别
  - 软文/广告嫌疑已判断
  - 推销动机已评估
  - **验证方法**：输出包含「利益动机」评分

- [ ] **实际可行性已评估**
  - 时间/成本合理性已判断
  - 隐藏门槛已识别
  - 幸存者偏差已检测
  - **验证方法**：输出包含「实际可行性」评分

- [ ] **可信度评分卡已生成**
  - 总体星级已评定
  - 合理部分已列出
  - 问题部分已列出（带严重程度）
  - 关键红旗已标注
  - 分维度评分表已生成
  - **验证方法**：输出符合「可信度评分卡」模板

- [ ] **可执行建议已提供**
  - 具体行动建议 ≥2 条
  - 风险提示明确（如适用）
  - 替代方案推荐（如适用）
  - **验证方法**：输出包含「建议」章节

#### 【任务：GitHub 项目评估】

- [ ] **项目基本信息已获取**
  - Stars/Forks 数量已获取
  - 最后更新时间已确认
  - 提交数量已统计
  - **验证方法**：输出包含「基本信息」表格

- [ ] **项目质量已评估**
  - 优点已列出（≥2 条）
  - 问题已列出（≥1 条）
  - 与现有方案对比已完成
  - **验证方法**：输出包含「优点」「问题」「对比」章节

- [ ] **最终评分已给出**
  - 星级已评定
  - 建议明确（使用/观望/放弃）
  - **验证方法**：输出包含「最终评分」

### 可选项（加分项）

- [ ] **多个可疑内容对比完成**
  - 找到类似案例
  - 对比分析表格已生成
  - **验证方法**：输出包含「对比案例」章节

- [ ] **深度验证已完成**
  - 联系作者/发布者（如可能）
  - 第三方权威验证（如可能）
  - **验证方法**：输出包含「深度验证」结果

### 失败处理

| 失败场景 | 处理路径 | 用户提示 |
|---------|---------|---------|
| 内容链接失效 | 使用缓存/快照 | ⚠️ 原链接失效，使用缓存版本 |
| 无法访问原始来源 | 降级评估 | ⚠️ 无法访问原始来源，评估可能不完整 |
| 信息严重不足 | 明确告知 | ❌ 信息不足，无法做出可信判断 |
| 证据冲突严重 | 标注不确定性 | ⚠️ 多源证据冲突，建议谨慎判断 |
