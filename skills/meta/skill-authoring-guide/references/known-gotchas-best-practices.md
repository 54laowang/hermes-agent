# Known Gotchas 编写最佳实践

## 核心原则

> **Known Gotchas = 实际失败案例的提炼，不是想象的问题**

## 写作规范

### 1. 必须来自真实失败

❌ **错误**（想象的问题）：
```markdown
- **内存溢出**: 数据量大时可能导致内存不足
  ```python
  # 建议分批处理
  for chunk in pd.read_csv(file, chunksize=1000):
      process(chunk)
  ```
```

✅ **正确**（实际失败案例）：
```markdown
- **A股数据延迟**: Tushare 免费接口有延迟
  ```python
  # 使用 pro_bar 获取实时数据（需要积分）
  df = pro.daily(ts_code='000001.SZ', start_date='20240101')
  
  # 免费接口延迟约 15 分钟
  # 付费接口可获取实时数据
  ```
```

**判断标准**：
- 你是否实际遇到过这个问题？
- 是否有具体的错误信息或异常行为？
- 是否有验证过的解决方案？

### 2. 必须包含错误示例和正确示例

❌ **错误**（只有描述）：
```markdown
- **技术指标钝化**: 趋势行情中震荡指标失效
```

✅ **正确**（对比示例）：
```markdown
- **技术指标钝化**: 趋势行情中震荡指标失效
  ```
  # 强趋势市场（涨跌幅 > 5%）
  # KDJ/RSI 会持续超买/超卖
  # 改用 MACD、均线等趋势指标
  ```
```

### 3. 按类别分组

**推荐分类**：

| 类别 | 内容 |
|------|------|
| **数据源问题** | 延迟、缺失、格式、时区 |
| **配置陷阱** | 参数不当、环境变量、权限 |
| **工具限制** | API 限流、超时、内存 |
| **逻辑错误** | 边界条件、类型转换、空值处理 |
| **平台特性** | A股特殊规则、时区、假期 |

**示例**：

```markdown
## ⚠️ Known Gotchas

### 数据源问题

- **A股数据延迟**: Tushare 免费接口有延迟
- **ST公司财务数据缺失**: 部分公司财报不完整
- **美股数据时区**: Yahoo Finance 使用美东时间

### 技术分析陷阱

- **艾尔德三重滤网参数不当**: 默认参数不适合所有市场
- **技术指标钝化**: 趋势行情中震荡指标失效
- **成交量陷阱**: A股缩量涨停不一定是买入信号

### 量化回测陷阱

- **过拟合风险**: 回测表现好 ≠ 实盘好
- **未来函数**: 使用了未来数据
- **手续费和滑点**: 回测未计入成本
```

### 4. 每个 Skill 至少 10 条

**最低标准**：
- 基础设施 Skills（如 `stock-data-acquisition`）：15-20 条
- 核心 Skills（如 `hermes-agent`）：20+ 条
- 辅助 Skills（如 `humanizer`）：10-15 条

**为什么是 10 条**：
- 少于 10 条说明 Skill 还没被充分使用
- 每次失败都应该补充 Gotcha
- 积累越多，未来失败越少

## Gotchas 来源

### 1. 实际执行失败

```
用户: 分析 000001.SZ
Agent: [执行] Tushare API 调用
       [错误] 免费接口延迟 15 分钟
       [补充] Gotcha: A股数据延迟
```

### 2. 用户纠正

```
用户: 为什么用周线分析A股？
Agent: 艾尔德三重滤网默认周线
用户: A股应该用日线/60分钟
Agent: [补充] Gotcha: 参数不当
```

### 3. 文档和社区

```
# 从官方文档提取已知问题
# 从 GitHub Issues 提取常见错误
# 从 Stack Overflow 提取解决方案
```

## Gotchas 模板

```markdown
### [类别名称]

- **问题标题**: 简短描述
  ```python
  # ❌ 错误示例
  错误代码
  
  # ✅ 正确示例
  正确代码 + 解释
  ```
  
  **原因**: 为什么会出现这个问题
  **解决**: 具体的解决方案
  **验证**: 如何验证是否解决
```

## 质量检查清单

- [ ] 按类别分组
- [ ] 总数 ≥ 10 条
- [ ] 没有重复或相似的 Gotchas
- [ ] 语言简洁，可操作
