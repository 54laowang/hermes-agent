## 💡 关键经验

### 1. 外移策略

| 内容类型 | 存储位置 | Memory保留 |
|----------|----------|-----------|
| 详细流程 | fact_store | 要点+指针 |
| 技术文档 | Skill references/ | 触发词 |
| API参数 | Skill references/ | 核心配置 |
| 项目进度 | 删除（完成后） | 最终状态 |

### 2. 触发词设计原则

**优秀触发词**：
- 包含关键词（用户会说的词）
- 包含核心功能（一句话说清楚）
- 简洁（< 100 chars）

**示例**：
```
✅ 淘股吧视角：「淘股吧怎么看」「游资动向」→ 6大心智模型
❌ 淘股吧视角Skill已生成：基于3067篇文章（2024-2026），包含6大心智模型...保存路径...
```

### 3. 优化时机判断

| 使用率 | 状态 | 行动 |
|--------|------|------|
| < 30% | 健康 | 无需操作 |
| 30-50% | 正常 | 定期检查 |
| 50-70% | 警告 | 计划优化 |
| > 70% | 紧急 | 立即优化 |

### 4. 保留vs删除决策树

```
内容是否重复？
├─ 是 → 删除Memory版本，保留fact_store/Skill版本
└─ 否 → 内容是否过时？
    ├─ 是 → 删除（项目已完成）或更新（技术已升级）
    └─ 否 → 内容是否过于详细？
        ├─ 是 → 简化为要点，详情外移
        └─ 否 → 保留
```
